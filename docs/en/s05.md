# Building a Knowledge Base and Support Portal

> "Publish only what you trust. Give customers the tools to solve problems themselves."

## The Problem

Most ticket queues are full of the same questions asked over and over. "I forgot my password." "I don't understand my invoice." "How do I authenticate to the API?" These are questions that, once documented, customers can answer themselves.

But scattered documentation, outdated articles, and inconsistent organization make things worse, not better. Customers find a help article, follow the instructions, and discover it describes an old version of the product.

DevRev's Knowledge Base (KB) lets you manage trusted information in a structured way and publish it through a branded Support Portal.

## The Solution

Register Articles in the KB, organize them into Collections, and publish them through the Support Portal. Three steps to a working customer self-service layer.

- **Article**: The atomic unit of knowledge. Always linked to a Part
- **Collection**: A category that groups Articles. Supports nesting
- **Support Portal**: The customer-facing public page. Provides ticket creation and KB search

## How It Works

### Creating KB Articles

There are three ways to add Articles:

| Method | Description |
|--------|-------------|
| Rich text editor | Write directly in the UI. Supports images, code blocks, and tables |
| URL link | Register an externally hosted page as a linked Article |
| File upload | Import a PDF or Word document as an Article |

**Required fields**

- **Part**: The product area this Article covers (required). States "which feature does this explain"
- **Owned by**: The Dev User responsible for this Article (required). Accountability for keeping it accurate

Article status follows three stages: **Draft → Published → Archived**. Only Published Articles appear in the Support Portal.

**The trusted-information principle**

Quality over quantity. Do not register articles that are outdated, unverified, or uncertain. Customers follow KB instructions as if they were correct. A wrong article causes more damage than no article at all.

### Bulk Import via URL Crawling

If you have an existing help center or documentation site, you can register the URL in the DevRev UI and let Turing (DevRev's AI) automatically pull in pages as knowledge.

**Steps**

1. Click **Setup 0/8** at the bottom of the left sidebar
2. On the Setup screen (`/settings/setup/platform`), open the **Connect your Knowledge Base** section
3. Under "Automatically load your knowledge base," click **Add URL Source**
4. Enter your URL in the modal, configure the settings, and click **Start training**

**Add URL Source modal settings**

| Field | Description |
|-------|-------------|
| URL | Enter your help center's top-level domain. All pages under that domain and its subdomains will be crawled |
| Link type | Choose URL or Sitemap |
| Override Regex | Optional: exclude URLs matching a specific pattern |
| Which part should the articles be linked to? | Select the Part to assign to the imported Articles |

**Notes**

- Providing a top-level domain causes Turing to read from all sub-domain pages
- Sites that block AI crawlers via `robots.txt`, or pages that rely on JavaScript rendering, may not be imported successfully
- Always review auto-imported Articles before publishing — irrelevant pages or inaccurate content may be included

### Designing Collections

Collections are the organizational structure for Articles. In the Support Portal, they become the top-level navigation.

**Where to create**: Settings > Support > Article Collections > +Collection

**Fields**

- **Title**: Display name of the Collection
- **Description**: Brief description shown in the portal
- **Parent**: Assign a parent Collection to create nested structure
- **Publish Collection**: Toggle on to make the Collection visible to customers

**Visibility rules**

For an Article to appear in the portal, all three conditions must be met:

1. Article status is **Published**
2. The Article's Collection has **Publish Collection** enabled
3. The Article's Visibility setting is **Public** or includes the target users

Collections support nesting. You can create navigation structures like "Getting Started > Account Setup > Two-Factor Authentication." Order within a Collection is controlled by drag-and-drop.

### Configuring the Support Portal

**Default URL**

```
https://support.devrev.ai/<yourcompany>
```

Custom domains (e.g., `help.example.com`) require filing a support ticket with DevRev.

**Appearance customization**

Settings > Plug & Portal > Portal Settings > Appearance:

| Setting | Description |
|---------|-------------|
| Logo / favicon | Upload your company logo |
| Color / theme | Brand colors and light/dark theme |
| Header / footer | Navigation links |
| Hero section | Main headline and background image on the portal home |

**What the portal provides**

- Ticket creation and status tracking
- KB search (supports both syntactic and semantic search)

### Language Settings

DevRev supports six languages:

| Language | Code |
|----------|------|
| English | en |
| Spanish | es |
| Japanese | ja |
| Dutch | nl |
| Portuguese | pt |
| Simplified Chinese | zh-CN |

**How multilingual support works**

Set up a parent Collection as a language folder for each language you support. Register translated Articles under the corresponding language Collection. The portal then switches display by language. You can preview the portal per language and device type.
