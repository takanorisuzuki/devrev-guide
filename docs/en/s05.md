# Understanding Customers with PLuG and Observe App

> "Talk to customers via live chat, watch their behavior with session replay. Combine both for deep customer understanding."

## The Problem

You cannot see what users are doing on your website or app. When a user contacts support, figuring out "what screen they were on and what they were doing" takes time. Analyzing user behavior requires a separate tool like Hotjar or FullStory, disconnected from your support platform.

This disconnect is a bottleneck for customer understanding.

## The Solution

DevRev provides two features as a set for customer understanding:

- **PLuG**: A live chat widget embedded in your website. It brings customer touchpoints directly into the platform
- **Observe App**: Provides session replay (recordings of user screen actions) and funnel analysis (conversion path analysis)

PLuG captures "what customers are saying." Observe App captures "what customers are doing." Combining both gives you deep context behind support tickets.

## How It Works

**PLuG Setup**

PLuG works by embedding a JavaScript tag in your website. Once embedded, you get:

- Live chat: Real-time messages from customers
- Knowledge base search: Automatic display of relevant FAQ articles
- Ticket creation: File tickets directly from chat

PLuG is configured from DevRev's Settings. Design customization (color, position, display timing) is also available.

**Observe App Features**

Observe App has two main capabilities:

1. **Session Replay**: Records mouse movements, clicks, scrolls, and page transitions. Personal information is masked
2. **Funnel Analysis**: Visualizes where users drop off at each screen

Session replay helps support agents see "what the user was doing" when handling a ticket. You can understand the reproduction steps of a reported problem by watching the replay.

Funnel analysis helps product teams identify "where users are dropping off."

**PLuG and Observe App Integration**

Tickets filed through PLuG automatically include the user's session information. Support agents can open a ticket and immediately watch the replay of what the user was doing before and after encountering the problem.

This automates what previously required manually cross-referencing Zendesk tickets with Hotjar sessions.
