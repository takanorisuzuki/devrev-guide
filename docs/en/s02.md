# Getting Started with DevRev

> "Master the big picture in your first 30 minutes. That changes the speed of everything after."

## The Problem

When you first log in to DevRev, the left sidebar is filled with menu items — but **none of the names look familiar**. You expect "Tickets" and find "Product Assist Tracker." You look for "Home" and find "Updates." You search for "Backlog" and find "Triage."

On top of that, items not shown by default must be added manually from Explore, with no explanation of how to do so in the UI.

Documentation is primarily in English, and there are many configuration options. Many people give up during this initial experience.

Users accustomed to Jira or Zendesk often find DevRev's navigation confusing because the naming conventions are fundamentally different.

## The Solution

What you need to do first comes down to four steps:

1. **Create a workspace**: In DevRev, a "workspace" is the organizational unit. Create one first
2. **Set up your profile**: Configure your account so you are recognized as a team member
3. **Customize the left sidebar**: Pin frequently used menus from Explore to set up your navigation
4. **Understand basic navigation**: Learn the left sidebar structure so you can access key screens without confusion

Completing these four steps in your first 30 minutes is the key to accelerating your learning.

## How It Works

### Default Left Sidebar Layout

When you log in to DevRev, the left sidebar already shows many items.

**Fixed menus (no section header)**

| Menu | Familiar equivalent | Actual role |
|------|--------------------|----|
| **Updates** | ≒ Home / Notifications | Activity feed and updates addressed to you |
| **My Tasks** | ≒ My task list | Work items assigned to you |
| **Explore** | Navigation manager | Pin and customize menus from here |

**Work section (developer-focused)**

| Menu | Jira equivalent |
|------|----------------|
| Issues | Backlog issues |
| Triage | Incoming issue triage |
| Now, Next, Later | Priority-based roadmap view |
| Sprint Boards | Sprint board |
| Dev360 | Development metrics dashboard |

**Product section**

| Menu | Role |
|------|------|
| Trails | Feature release tracking |
| Parts | Product structure definition (product / feature / component) |
| Roadmap | Roadmap view |
| Release Tracker | Release management |

**Customer section (support-focused)**

| Menu | Zendesk equivalent |
|------|-------------------|
| Product Assist Tracker | ≒ Tickets (product inquiries and requests) |
| Accepted Ticket Tracker | ≒ Active ticket queue |

**Data section**

| Menu | Role |
|------|------|
| Notebook | Notes and documents |

> **Key point**: Names like "Tickets," "Home," and "Backlog" do not exist as-is. Getting comfortable with DevRev's naming system is the first hurdle.

### How to Customize the Left Sidebar (Pinning from Explore)

To add items to the left sidebar, use Explore.

1. Click **"Explore"** in the left sidebar
2. Hover over the view you want to add
3. Click the **pin icon (📌)** that appears
4. The view is added to your left sidebar

To unpin, hover over the same view and click the **unpin icon**.

#### Explore categories

| Category | Contents |
|---------|------|
| **Stock Views** | Default views provided by DevRev (My Issues, etc.) |
| **My Views** | Views you have created |
| **Shared** | Views shared with your organization or team |

> **Recents**: The five most recently opened unpinned views appear automatically in the sidebar. These include views opened from Explore or accessed via shared links.

### Add the menus you will use in this course from Explore

The following menus are not in the default sidebar but are used frequently throughout this course (s01–s14). Add them all now to avoid interruptions later. Search for each by name in the Explore search bar, then pin the result from Stock Views.

| Menu | Search term | First used | Purpose |
|------|------------|-----------|---------|
| **Tickets** | ticket | s04+ | Support ticket management |
| **Conversations** | conv | s06+ | PLuG live chat |
| **Inbox** | inbox | s06+ | Unified view for chat, email, and customer messages |
| **Accounts** | account | s03+ | Company-level customer management (CRM) |
| **Contacts** | contact | s03+ | Individual contact management (CRM) |

#### Save a filter on Tickets to show only your tickets

The Tickets view shows all tickets by default. To see only tickets assigned to you, save a filter.

1. Open the Tickets view
2. Select **"Assignee" → your account** in the filter bar
3. Click **"Save"** in the top right

### Other basic features

DevRev provides search and keyboard shortcuts. The search bar at the top lets you search across tickets, issues, and parts.

When creating a workspace, you configure the following:

| Setting | Description |
|---------|-------------|
| Workspace name | Organization or project name |
| Timezone | Affects SLA policy calculations |
| Default language | UI and template language |

## What You Will Learn

**Before this session:**
- You created an account but do not know what to do after logging in
- The sidebar has menus but none of the names are familiar
- You do not know how to customize navigation by pinning from Explore
- You are unaware of search and shortcuts

**After this session:**
- Your workspace is created and profile is set up
- You can pin frequently used views from Explore and configure your own sidebar
- You can explain the role of each sidebar section (Work / Product / Customer / Data)
- You can quickly access objects using the search bar
