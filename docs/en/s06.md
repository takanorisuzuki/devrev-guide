# Managing the Development Cycle

> "Create an Issue from a Ticket, link it to a Part, track it through release. Support and engineering connected in one tool."

## The Problem

Bug reports from customers take days to reach engineers. Even when they do, the link to the original ticket is lost, and when the fix ships, nobody remembers to notify the customer. Manually linking Jira issue numbers to Zendesk ticket numbers breaks down at scale.

## The Solution

DevRev's Build App lets you create Issues directly from support tickets, link them to Parts, and track them through to release. The entire flow from Ticket to Issue to Part to Release stays within a single platform.

## How It Works

**Ticket-to-Issue Linking**

Executing "Create Issue" from a support ticket creates an Issue linked to that ticket. This enables:

- When an Issue is resolved, the original ticket creator can be automatically notified
- Multiple Tickets reporting the same problem can be consolidated under a single Issue
- Issue progress is visible from the Ticket side

**Issue Management**

DevRev Issues have these attributes:

| Attribute | Description |
|-----------|-------------|
| Type | Bug / Feature Request / Task, etc. |
| Priority | Blocker / High / Medium / Low |
| Part | Linked Product/Feature |
| Assignee | Responsible person |
| Sprint | Associated sprint |
| Stage | Open / In Progress / In Review / Done |

**Sprint Management**

Build App lets you define sprints (development cycles). Assign Issues to sprints and track progress with burndown charts.

**Release Management**

Releases are managed as collections of Issues. Automatic release note generation is possible, summarizing all Issues included in a release.

This complete flow lets you track "when the problem a customer reported will be fixed in a release."

## What You Will Learn

**Before this session:**
- Support and development are managed in separate tools
- Ticket-to-Issue linking is manual
- Understanding what a release includes takes time

**After this session:**
- You can create an Issue from a Ticket and link it to a Part
- You can assign Issues to sprints and track progress
- You understand the release management flow and can use auto-generated release notes
