# Understanding Customers with PLuG and Observe App

> "Talk to customers via live chat, watch their behavior with session replay. Combine both for deep customer understanding."

## The Problem

You cannot see what users are doing on your website or app.

When a user contacts support, figuring out "what screen they were on and what they were doing" takes time.

Analyzing user behavior requires a separate tool like Hotjar or FullStory, disconnected from your support platform.

This disconnect is a bottleneck for customer understanding.

## The Solution

DevRev provides two features as a set for customer understanding:

- **PLuG**: A live chat widget embedded in your website. It brings customer touchpoints directly into the platform
- **Observe App**: Provides session replay (recordings of user screen actions) and funnel analysis (conversion path analysis)

PLuG captures "what customers are saying."

Observe App captures "what customers are doing."

Combining both gives you deep context behind support tickets.

## How It Works

### PLuG Setup

PLuG works by integrating the **PLuG SDK** into your website — the same approach used by Intercom, Zendesk widget, and similar tools. Add a few lines of code to get started.

**Step 1: Load the script**

Add the following script tag to your `<head>` or before the closing `</body>` tag:

```html
<script src="https://plug-platform.devrev.ai/static/plug.js" async></script>
```

**Step 2: Initialize the SDK**

After page load, call `window.plugSDK.init()`. The `app_id` is found in DevRev Settings > PLuG Settings > **Your Unique App ID**.

```javascript
// Basic initialization
window.plugSDK.init({
  app_id: "YOUR_APP_ID",
});
```

For React applications, call it inside `useEffect`:

```javascript
useEffect(() => {
  window.plugSDK.init({
    app_id: "YOUR_APP_ID",
  });
}, []);
```

Once initialized, a chat launcher appears in the bottom-right corner.

**Step 3: Enable session recording (recommended)**

To integrate with Observe App, explicitly enable session recording:

```javascript
window.plugSDK.init({
  app_id: "YOUR_APP_ID",
  enable_session_recording: true,
  session_recording_key: "YOUR_SESSION_RECORDING_KEY",
});
```

The `session_recording_key` is available in Settings > PLuG Settings.

**Key `init()` parameters**

| Parameter | Description | Type |
|-----------|-------------|------|
| `app_id` | Unique PLuG identifier (**required**) | string |
| `enable_session_recording` | Enable session recording | boolean |
| `session_recording_key` | Session recording account identifier | string |
| `session_token` | User identification token (see below) | string |
| `enable_default_launcher` | Show/hide the default launcher icon | boolean |
| `disable_plug_chat_window` | Disable the chat window | boolean |
| `primary_text_color` | Launcher and button text color (hex: `#f0f0f0`) | string |

**User Identification (Verified Identification)**

PLuG works for anonymous users, but linking to a RevUser enables access to past conversation history and account context.

1. Generate an **Application Access Token (AAT)** from Settings > PLuG Tokens
2. On your backend, use the AAT to call the DevRev API (`auth-tokens.create`) and obtain a session token
3. Pass the `session_token` to `init()` on the frontend

```javascript
window.plugSDK.init({
  app_id: "YOUR_APP_ID",
  session_token: "SESSION_TOKEN_FROM_BACKEND",
});
```

This links chats and tickets to a specific RevUser, giving support agents immediate context.

**Logout handling**

When a user logs out, call `shutdown()` to clear conversation data and stop session recording:

```javascript
await window.plugSDK.shutdown();
```

Re-run `init()` when the user logs in again.

**PLuG capabilities**

- **AI-powered chat widget**: Real-time customer messages with AI auto-responses
- **Knowledge base search**: Automatic display of relevant FAQ articles
- **Search Agent**: Dedicated search UI via `initSearchAgent()`
- **Ticket creation**: File tickets directly from chat
- **Nudges**: Outbound messages from the widget (proactive guidance and engagement prompts)

PLuG is configured from DevRev Settings > PLuG Settings. Design customization (launcher logo, position, colors, page visibility rules) is available. Use **View in Playground** at the bottom of the settings page to preview.

**Supported platforms**

PLuG SDK supports more than just web — it works on mobile apps too.

| Platform | Package / SDK | Notes |
|----------|--------------|-------|
| Web | PLuG SDK (JavaScript) | `init()` method described above |
| iOS | DevRev SDK (Swift) | Minimum deployment target iOS 15.0 |
| Android | `ai.devrev.sdk:devrev-sdk` | Minimum API level 24 |
| Flutter | `devrev_sdk_flutter` | Dart SDK 3.7.0+ |
| React Native | DevRev React Native SDK | — |

All SDKs share common features: chat, session recording, user identification, and data masking. Embedding PLuG in mobile apps provides the same support experience as web.

### Observe App Features

Observe App is a suite of analytics features for understanding "what users are doing." It works in conjunction with PLuG SDK's session recording.

**1. Session Replay**

Records mouse movements, clicks, scrolls, and page transitions. Sessions are integrated into DevRev's knowledge graph — you can navigate directly from tickets or conversations to the relevant session.

Recordings also capture:

- **Page transitions**: All pages visited and their order
- **Click / input events**: Interactions with UI elements
- **Network calls**: API requests and responses (including errors)
- **Console errors**: Real-time JavaScript error capture
- **Experience quality issues**: Dead clicks (no response) and rage clicks (rapid repeated clicks) are auto-detected

> **Data masking**: Add the CSS class `rr-mask` to sensitive UI elements (web), or use the `markSensitiveViews` method in mobile SDKs to automatically mask personal information.

**2. Funnel Analysis**

Visualizes where users drop off. The funnel builder uses custom events to define arbitrary steps.

- Fire custom events with `plugSDK.trackEvent()`
- Arrange events as steps in the funnel builder
- Click on drop-off segments to drill down into the relevant session list

```javascript
// Custom event example
window.plugSDK.trackEvent("checkout_started", {
  plan: "enterprise",
  source: "pricing_page",
});
```

**3. Session Analytics Dashboard**

A built-in dashboard that requires no setup. Simply enable session recording and the following metrics are automatically tracked:

- Page views and unique users over time
- Pages with the most dead clicks
- Average session duration
- Drill-down from any widget to the session list

Custom dashboards can also be created (covered in detail in s09).

**4. Cross-domain Session Tracking**

Track user journeys across multiple domains as a single session.

```javascript
// Get session details on Domain A
const { sessionId, tabId } = window.plugSDK.getSessionDetails();

// Pass to Domain B and include during initialization
```

### PLuG and Observe App Integration

Tickets filed through PLuG automatically include the user's session information. Support agents can open a ticket and immediately watch the replay of what the user was doing before and after encountering the problem.

This integration enables:

- **Conversation → Session**: Navigate directly from a chat conversation to the related session replay
- **Funnel drop-off → Session**: Identify and investigate sessions where users dropped off
- **Dead Click → Session**: Find sessions with UI issues and connect them to fixes

This automates what previously required manually cross-referencing Zendesk tickets with Hotjar sessions.
