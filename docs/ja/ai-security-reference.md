# AI/LLMセキュリティ・データ保護リファレンス

お客様社内でのAI利用申請・セキュリティ審査に必要な情報をまとめたリファレンスです。

[s01](/ja/s01) で触れたAIアーキテクチャの補足として、データ保護とコンプライアンスの観点を整理しています。

---

## 前提: DevRevのAI機能

DevRevはAI-nativeプラットフォームとして設計されており、AI機能はサービス内に完全統合されています。
Microsoft CopilotやChatGPTなど外部生成AIサービスを別途契約・連携する必要はありません。

主なAI機能:

- **Computer（対話型AIエージェント）** - SQL分析、セマンティック検索、アクション実行、チケット自動分類・要約
- **PLuG（カスタマー向けAI）** - エンドユーザー向けAIチャットボット、ナレッジベースからの回答生成

## データ処理アーキテクチャ

ComputerはText2SQL + RAGのハイブリッドアーキテクチャを採用しています。
詳細は [architecture.md](/ja/architecture) の「Computer Memory アーキテクチャ」も参照してください。

### 処理フロー

```
ユーザーの自然言語クエリ
  → Text2SQL（ナレッジグラフへのSQLクエリ変換・実行）  ← DevRev自社インフラ
  → セマンティック検索（RAG）で非構造化データを補完     ← DevRev自社インフラ
  → 権限フィルタリング済みコンテキストを構築           ← DevRev自社インフラ
  → 最小限のコンテキストチャンクのみLLMに送信         → 外部LLM推論
  → 生成された回答をユーザーに返却（監査ログに記録）
```

### 処理場所の一覧

| 処理項目 | 実行場所 | 詳細 |
|---------|---------|------|
| Text2SQL（構造化データ取得） | DevRev自社インフラ | 自然言語→SQL変換、ナレッジグラフからの取得。全処理がDevRev内で完結 |
| セマンティック検索・埋め込み生成 | DevRev自社インフラ | オープンソースモデル使用。ベクトルインデックスもDevRev内に保存 |
| ナレッジグラフ | DevRev自社インフラ | お客様データはDevRevのセキュアなクラウド内に保持 |
| LLM推論（生成） | 外部LLMプロバイダー | 権限フィルタリング済みの最小限コンテキストのみ送信。Zero Data Retention契約 |
| 監査ログ | DevRev自社インフラ | 全AI操作を記録。完全な可視性を提供 |

5処理中4処理がDevRev管理インフラ内で完結します。
外部LLMには権限チェック済みの最小限コンテキストのみが送信されます。

### LLMプロバイダーとモデル非依存設計

DevRevは特定のLLMベンダーにロックインしないモデル非依存（model-agnostic）アーキテクチャを採用しています。

- 知識の蓄積はDevRev側のナレッジグラフに保持され、LLMは「推論エンジン」として差し替え可能
- モデル選択はテナントレベルで設定可能
- 顧客が承認したモデル（例: Azure OpenAI のみ、自社ホストモデルのみ）に限定する構成にも対応可能
- より優れたモデルが登場した場合、ナレッジグラフに変更なく即座に切替が可能

すべてのLLMプロバイダーとの契約において、
**エンタープライズ契約（送信データがモデルトレーニングに使用されない）** が前提です。

### Computer App（デスクトップ版）の場合

Computer App（デスクトップ）では、ローカルのClaude CodeエンジンからAnthropic APIにLLM推論リクエストが直接送信されます。
DevRevプラットフォームへはMCPプロトコル経由でデータ取得のみ行います。

| 項目 | Computer（Web版） | Computer App（デスクトップ版） |
|------|------------------|---------------------------|
| LLM推論 | DevRev管理インフラ経由 | ローカルからAnthropic API（Claude）に直接接続 |
| データアクセス | DevRevサーバーサイドで完結 | MCPプロトコル経由でDevRevプラットフォームに接続 |
| 実行環境 | ブラウザ（SaaS） | ユーザーのローカルマシン |
| ローカルファイルアクセス | 不可 | 可能（ユーザー許可のもと） |

デスクトップ版のセキュリティ保護:

- **学習不使用**: Anthropic APIもエンタープライズ契約により、送信データのトレーニング利用は禁止
- **DevRevへの認証**: MCPサーバーへの接続はOAuthまたはPAT（Personal Access Token）で認証。テナント分離が維持される
- **データの最小送信**: DevRevのMCPサーバーは権限フィルタリング済みのデータのみを返却
- **監査ログ**: DevRevプラットフォーム側の操作（MCP経由のデータアクセス）は監査ログに記録
- **ローカルデータの保護**: ローカルファイルへのアクセスはユーザーの明示的な許可が必要。DevRevプラットフォームにローカルファイルが自動送信されることはない

## 「学習に使用しない」ことの3層保護

お客様データがLLMのトレーニングに使用されないことは、以下の3層で担保されています。

### 第1層: LLMプロバイダーとのエンタープライズ契約

DevRevが利用するすべてのLLMプロバイダーとの契約には、以下が含まれます:

- 送信データをモデルトレーニングに使用しないことの明示的な禁止
- **Zero Data Retention (ZDR)** - 推論に送信されたデータはプロバイダー側に保存されない
- これらはDevRev固有ではなく、各プロバイダーの**標準的なエンタープライズ契約条件**として公開されている

### 第2層: DevRevのDPA（データ処理契約）

- DevRevのDPAにて、AI処理シナリオに対する保護措置が明記
- サブプロセッサーが顧客データを学習に使用することを**契約上明示的に禁止**
- DPAは https://devrev.ai/legal/dpa で入手可能

### 第3層: アーキテクチャ上のデータ最小化

- LLMに渡されるのは**権限フィルタリング済みの、クエリに関連するコンテキストチャンクのみ**
- 生の顧客レコードや大量データエクスポートがLLMに送られることはない
- 検索・埋め込み・ベクトルインデックスは**DevRev自社インフラ上で完結**

## データ保存リージョンと越境移転

### 利用可能リージョン

DevRevはAWS（プライマリ）およびGCPにホストされており、以下のリージョンから顧客が選択できます:

| リージョン | ロケーション |
|-----------|------------|
| US East | N. Virginia |
| EU Central | Frankfurt |
| AP South | Mumbai |
| AP Southeast | Singapore |
| AP Southeast | Sydney |
| AP Northeast | Japan |

データは顧客が選択したリージョンにローカライズされ、
**選択したリージョン外にデータが移動することはありません**。
日本リージョンを選択すれば、データは日本国内に保持されます。

### 日本企業向けの越境データ移転の根拠

- **日本リージョン対応**: 日本国内にデータを保持する構成が可能
- DevRevのDPAにはEU標準契約条項（SCCs）が含まれ、越境データ移転の法的根拠を提供
- 個人情報保護法における外国にある第三者提供規制に対しては、日本リージョン選択により国内保持が可能。
加えてDPA上の契約条項で法的根拠を提供
- LLM推論に送信されるデータは権限フィルタリング済みの最小限コンテキストのみであり、
Zero Data Retention により推論後に即時破棄される

## DevRev従業員によるデータアクセス

DevRev従業員が顧客データに対して持つアクセスは、以下のように厳格に制限されています:

| 項目 | 詳細 |
|------|------|
| デフォルトのアクセス権 | **無効**。本番環境の顧客データへのアクセスはデフォルトで禁止 |
| アクセス付与の条件 | 承認プロセスを経て、必要な場合にのみ一時的に付与 |
| 承認者 | インフラストラクチャ・セキュリティチームがケースバイケースでレビュー |
| アクセス期間 | 一時的（タスク完了後に自動失効） |
| 監視 | Datadog、Prometheus、Grafanaによる継続的モニタリング |
| 背景調査 | 従業員に対するバックグラウンドチェックを実施 |

要約: DevRev従業員は顧客データへの自由なアクセス権を持ちません。
全アクセスは最小権限の原則に基づき、承認・時限・監査されています。

## データ削除と保持

### 契約終了後のデータ削除

- サービス契約終了日またはユーザーアカウント削除日のいずれか早い方から、**30営業日以内**にすべてのコピーを含めて削除
- 個別のデータ削除要求は、ユーザーダッシュボード/データ削除機能から即時実行可能

### データ保持期間

| データ種別 | 保持期間 |
|-----------|---------|
| アクティブデータ | サービス利用期間中。削除要求から30日以内に削除 |
| バックアップ | 12ヶ月間保持 |
| 監査ログ | 最低7年間保持 |

## インシデント通知プロセス

DevRevはセキュリティインシデント発生時に以下のプロセスで顧客に通知します:

- **文書化されたインシデント対応手順**が整備・テスト済み
- インシデントは重大度に基づいて分類され、分類レベルがチケットに記録される
- 本番環境の停止またはビジネスに影響する重大な障害に対する初動レスポンスSLAは **60分**
- セキュリティインシデント通知はDPA条項に基づき、
影響を受ける顧客に速やかに行われる
- **ステータスページ**: https://status.devrev.ai/ でリアルタイムのサービス稼働状況を確認可能

> **個人情報保護法の72時間報告義務への対応**: DevRevはインシデント発見後、DPAに基づき不当な遅滞なく顧客に通知します。
> 通知には事象の性質、影響範囲、講じた対策が含まれます。

|------|------|

## AIガバナンスと安全対策

### プロンプトインジェクション対策

DevRevは自社開発のガードレールフレームワークで多層防御を実装しています:

**入力ガードレール（LLMに到達する前に適用）:**

- PII/PHIの検出・マスキング
- 有害コンテンツ・スコープ外リクエストのブロック
- プロンプトインジェクション検知・無効化
- 入力バリデーション（adversarial input のフィルタリング）

**出力ガードレール（ユーザーに返却する前に適用）:**

- 事実精度の検証（ナレッジソースとのクロスチェック）
- センシティブデータの漏洩防止
- ツール呼び出しの妥当性検証（実行前に許可確認）

### ハルシネーション（幻覚）対策

- **RAGアーキテクチャ**: LLMの内部学習データではなく、組織固有のナレッジから検索された情報のみに基づいて回答を生成
- **引用の提示**: 回答には参照元が常に含まれ、トレーサビリティを確保
- **Human-in-the-Loop**: 高リスクなアクションには人間による確認フローを組み込み可能
- **決定論的バリデーション**: ビジネスルールに基づくハードコードされた検証ステップが
LLMの解釈に依存せず実行される

### AIの操作の可逆性

- 全AI操作はバージョン管理され、問題があれば即座に差し戻しが可能
- 本番環境への反映前に確認ワークフローを設定可能（ステージング → 承認 → 反映）

### 権限継承

- AIの回答範囲は**そのユーザーのアクセス権限に限定**される
- ソースシステムのACL（アクセス制御リスト）をナレッジグラフが自動継承
- ユーザーコンテキスト内でLLMが実行され、
昇格されたアクセスやバックドアアクセスは付与されない

## 想定Q&A（セキュリティ審査対応）

**Q. 弊社のデータでAIが学習（トレーニング）しますか？**

いいえ。DevRevはお客様データによるモデルのトレーニング・ファインチューニングを一切行いません。
LLMプロバイダーとのエンタープライズ契約にはZero Data Retention（ZDR）ポリシーが含まれており、
推論に送信されたデータはプロバイダー側に保存もトレーニング利用もされません。

**Q. データは日本国内に保存されますか？**

はい。日本リージョンに対応しており、日本国内にデータを保持する構成が可能です。
データは顧客が選択したリージョンにローカライズされ、選択リージョン外に移動することはありません。

**Q. データは外部に送られますか？**

LLM推論のために、権限フィルタリング済みの最小限のコンテキストチャンクのみが外部LLM APIに送信されます。
生の顧客レコードや大量データが送られることはありません。
検索・埋め込み処理はDevRev自社インフラ内で完結しています。
LLMプロバイダーとのZDR契約により、送信データは推論後に即時破棄されます。

**Q. どのLLMを使用していますか？**

モデル非依存（model-agnostic）設計であり、特定ベンダーにロックインしません。
テナントレベルでモデル選択が設定可能で、お客様が承認したモデルのみを利用する構成にも対応できます。
すべてのプロバイダーとの契約でトレーニング不使用が担保されています。

**Q. DevRevの社員はデータを見ることができますか？**

本番環境の顧客データへのアクセスはデフォルトで無効です。
必要な場合にのみ、承認プロセスを経て一時的に付与されます。
アクセスはインフラ・セキュリティチームによるケースバイケースのレビューが必要で、
全アクセスは監査ログに記録されます。

**Q. 契約終了後のデータ削除はいつ行われますか？**

サービス契約終了日またはユーザーアカウント削除日のいずれか早い方から、
30営業日以内にすべてのコピーを含めて削除されます。

**Q. インシデント発生時の連絡はいつ来ますか？**

文書化されたインシデント対応手順に基づき、セキュリティインシデント発見後、
DPA条項に従い不当な遅滞なく影響を受ける顧客に通知します。
サービス停止・障害はステータスページ（status.devrev.ai）でリアルタイムに公開されます。

**Q. 他社のデータと混ざりませんか？**

テナント分離がソフトウェア・ハードウェア両レベルで厳密に行われており、
他テナントのデータが混在することはありません。
全データベースクエリにテナント識別子が含まれ、アクセスが制限されます。
AIの回答もユーザーのアクセス権限の範囲に限定されます。

**Q. Computer App（デスクトップ）ではデータはどこに送られますか？**

ローカルのClaude CodeエンジンからAnthropic APIにLLM推論リクエストが送信されます。
DevRevプラットフォームへはMCPプロトコル経由でデータ取得のみ行います。
Anthropic APIもエンタープライズ契約により、送信データのトレーニング利用は禁止されています。

**Q. 「学習に使わない」ことを証明する書類はありますか？**

はい。(1) DevRevのDPA（データ処理契約）、(2) サブプロセッサー一覧、(3) SOC 2 Type IIレポートの3点セットを提出可能です。
加えて、LLMプロバイダー各社の公式規約でもエンタープライズ契約におけるトレーニング不使用が明記されています。

**Q. AIの操作はどのように記録されますか？**

全AI操作（誰が何を質問し、どのデータを参照し、どの回答を得たか）を監査ログに完全記録しています。
監査ログは最低7年間保持され、J-SOX等のコンプライアンス監査に対応します。

**Q. プロンプトインジェクション等の攻撃への対策はありますか？**

自社開発のガードレールフレームワークにより、入力段階でのプロンプトインジェクション検知・ブロック、
出力段階での事実精度検証・センシティブデータ漏洩防止を多層で実装しています。
加えて、権限モデルによりAIがアクセスできるデータはそのユーザーの権限範囲に限定されるため、
仮にインジェクションが成功しても未認可データへのアクセスは構造的に防止されます。
