# Snap-inを自作する

> "npmパッケージのように公開・配布できる。ローカルテストから本番デプロイまでを一気通貫で学ぶ。"

## 問題

マーケットプレイスにない機能を自社で実装したい。たとえば「特定の条件でチケットが作成されたら社内の独自システムにデータを送信する」「毎朝9時に未対応チケットのサマリーをSlackに投稿する」など、自社固有の要件を満たすカスタム連携が必要になる。

## 解決策

DevRev CLIとTypeScriptを使ってSnap-inを開発する。Snap-inはnpmパッケージと同じようにバージョン管理・公開・配布できる構造になっている。

## 仕組み

**開発環境のセットアップ**

Snap-in開発には以下のツールが必要。

| ツール | 用途 |
|--------|------|
| Node.js | ランタイム |
| DevRev CLI (`devrev`) | プロジェクト作成、テスト、デプロイ |
| TypeScript | Snap-inのロジック記述 |

DevRev CLIのインストールは以下のコマンドで行う。

```bash
npm install -g @devrev/cli
```

**プロジェクト構造**

DevRev CLIでSnap-inプロジェクトを作成すると、以下の構造が生成される。

```
my-snap-in/
  ├── manifest.yaml        # Snap-inの定義ファイル
  ├── src/
  │   └── functions/
  │       └── handler.ts   # Function handler
  ├── package.json
  └── tsconfig.json
```

**manifest.yaml**

manifest.yamlはSnap-inの設計図である。以下の要素を定義する。

- **version**: Snap-inのバージョン
- **functions**: 実行するFunction群の定義
- **event_sources**: トリガーとなるイベント
- **keyrings**: 必要な外部認証情報

**Function handler**

Function handlerはSnap-inのビジネスロジックを記述する場所である。イベントを受け取り、処理を実行し、結果を返す。

```typescript
import { client } from "@devrev/typescript-sdk";

export async function handleEvent(event: any) {
  // イベントの内容に応じた処理を記述
  const devrevClient = client.setupBeta({
    endpoint: event.context.dev_oid,
    token: event.context.secrets.service_account_token,
  });

  // 例: チケットにコメントを追加
  await devrevClient.timelineEntriesCreate({
    object: event.payload.work_id,
    body: "Snap-inからの自動コメント",
    type: "timeline_comment",
  });
}
```

**ローカルテスト**

DevRev CLIにはローカルテスト機能がある。実際にデプロイする前に、手元の環境でFunctionの動作を確認できる。

```bash
devrev snap_in test
```

**デプロイフロー**

1. `devrev snap_in draft`: ドラフトとしてアップロード
2. DevRevのUI上でSnap-inを有効化
3. テスト環境で動作確認
4. 本番ワークスペースにインストール

## このセッションで学ぶこと

**始める前の状態：**
- Snap-in開発の経験がない
- manifest.yamlの構造を知らない
- ローカルテストの方法を知らない

**このセッション後の状態：**
- DevRev CLIでSnap-inプロジェクトを作成できる
- manifest.yamlの主要要素を説明できる
- Function handlerを記述し、ローカルテストとデプロイを実行できる
