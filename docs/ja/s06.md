# PLuGとObserve Appで顧客を理解する

> "ライブチャットで顧客と話し、セッションリプレイで行動を見る。2つを組み合わせて顧客を深く理解する。"

## 問題

Webサイトやアプリ上でユーザーがどんな行動をとっているかが見えない。

ユーザーが問い合わせてきたとき、「どの画面で何をしていたか」を把握するのに時間がかかる。

ユーザー行動の分析にはHotjarやFullStoryを別途導入し、サポートツールとは切り離された環境で確認する必要がある（日本ではPtengine・UserHeat・SiTestなどが同カテゴリに当たる）。

この分断が、顧客理解のボトルネックになっている。

## 解決策

DevRevには顧客理解のための2つの機能がセットで提供されている。

- **PLuG**: Webサイトに埋め込むライブチャットウィジェット。顧客との接点を直接プラットフォーム内に取り込む
- **Observe App**: セッションリプレイ（ユーザーの画面操作を録画）とファネル分析（コンバージョン経路の分析）を提供する

PLuGで「顧客が何を言っているか」を把握し、Observe Appで「顧客が何をしているか」を把握する。

この2つを組み合わせることで、サポートチケットの背景を深く理解できる。

## 仕組み

**PLuGの導入**

PLuGはJavaScriptタグをWebサイトに埋め込むことで利用できる。他のチャットウィジェット（IntercomやZendeskのウィジェットなど）と同じ方法で、`<head>` または `<body>` 終端に数行追加するだけだ。

```html
<script
  src="https://plug-platform.devrev.ai/static/plug.js"
  data-app-id="YOUR_APP_ID"
  crossorigin="anonymous"
  async
></script>
```

`data-app-id` はDevRevのSettings → PLuG画面で確認できる。タグを配置してページを読み込むと、右下にチャットアイコンが表示される。

埋め込み後、以下の機能が使える。

- ライブチャット: 顧客からのリアルタイムメッセージ
- ナレッジベース検索: FAQや記事の自動表示
- チケット作成: チャットから直接チケットを起票

PLuGの設定はDevRevのSettings画面から行う。デザインのカスタマイズ（色、位置、表示タイミング）も可能。

**Observe Appの機能**

Observe Appは2つの主要機能を持つ。

1. **セッションリプレイ**: ユーザーのマウス操作、クリック、スクロール、画面遷移を録画する。個人情報はマスキングされる
2. **ファネル分析**: ユーザーがどの画面でどれだけ離脱しているかを可視化する

セッションリプレイは、サポート担当者がチケット対応時に「ユーザーが何をしていたか」を確認するために使う。「ユーザーが報告した問題の再現手順」を、リプレイを見ることで把握できる。

ファネル分析は、プロダクトチームが「どこでユーザーが離脱しているか」を特定するために使う。

**PLuGとObserve Appの連携**

PLuGから起票されたチケットには、そのユーザーのセッション情報が自動的に紐付く。サポート担当者はチケットを開くだけで、ユーザーが問題に遭遇した前後の操作をリプレイで確認できる。

これは、従来のZendesk+Hotjarの組み合わせでは手動で行っていた作業を自動化するものである。

## このセッションで学ぶこと

**始める前の状態：**
- PLuGの導入方法を知らない
- セッションリプレイの活用経験がない
- 顧客行動分析とサポートツールが分離している

**このセッション後の状態：**
- PLuGのJavaScriptタグを埋め込み、ライブチャットを動作させられる
- セッションリプレイでユーザーの操作を確認できる
- PLuGとObserve Appの連携による効率化を説明できる

## 試してみる

テストページにPLuGを埋め込んで動作確認しよう。

```
【PLuG導入チェックリスト】

□ 1. DevRevのSettingsからPLuGのJavaScriptタグを取得した
□ 2. テスト用HTMLページにタグを埋め込んだ
□ 3. ページを開いてPLuGウィジェットが表示されることを確認した
□ 4. PLuGからメッセージを送信し、DevRevのInboxに届くことを確認した
□ 5. Observe Appでセッションリプレイを確認した
□ 6. PLuGから起票したチケットにセッション情報が紐付いていることを確認した
```
