# Snap-inとマーケットプレイスを活用する

> "まずはマーケットプレイスの既製Snap-inを活用する。それが最速の拡張方法。"

## 問題

DevRev単体では完結しない業務がある。Slackでの通知、GitHubとのコード連携、Jiraからのデータ移行。既存のツールエコシステムとDevRevをつないで使いたいが、自前で連携を開発する工数は取れない。

## 解決策

DevRevのマーケットプレイスには、主要ツールとの連携を実現する既製のSnap-inが用意されている。インストールするだけで連携が始まるため、開発工数ゼロで拡張できる。

Snap-inには2種類ある。

- **既製Snap-in**: マーケットプレイスからインストールして使うもの
- **カスタムSnap-in**: 自分でTypeScriptで開発するもの（s12で扱う）

まずは既製Snap-inの活用から始めるのが最速の拡張方法である。

## 仕組み

**マーケットプレイスの構成**

DevRevのマーケットプレイスでは、カテゴリ別にSnap-inを検索できる。

| カテゴリ | 主なSnap-in例 |
|---------|-------------|
| Communication | Slack, Microsoft Teams |
| Source Control | GitHub, GitLab, Bitbucket |
| Project Management | Jira (AirSync) |
| CRM | Salesforce, HubSpot |
| Monitoring | PagerDuty, Datadog |

**Snap-inのアーキテクチャ概念**

Snap-inは以下の要素で構成される。

- **Functions**: Snap-inのロジックを実行するサーバーレス関数
- **Event Sources**: トリガーとなるイベントの定義
- **Keyring Connections**: 外部サービスとの認証情報の管理

**Keyring Connections**

外部サービスとの連携には認証情報が必要になる。DevRevではKeyring Connectionsという仕組みで、APIキーやOAuthトークンを安全に管理する。

Snap-inのインストール時に、必要なKeyring Connectionの設定を求められる。設定は一度行えば、Snap-inが自動的に認証情報を利用する。

**AirSyncフレームワーク**

AirSyncはDevRevが提供する双方向同期フレームワークである。Jiraなどの外部プロジェクト管理ツールとDevRevの間で、データを双方向に同期する。

AirSyncの特徴は以下の通り。

- 双方向同期: DevRevで変更した内容が外部ツールに反映され、その逆も同様
- マッピング設定: 外部ツールのフィールドとDevRevのフィールドの対応を定義する
- コンフリクト解決: 両方で同時に変更された場合のルールを設定する

## このセッションで学ぶこと

**始める前の状態：**
- Snap-inの概念を知らない
- マーケットプレイスの存在を知らない
- 外部ツール連携を自前で開発しようとしている

**このセッション後の状態：**
- マーケットプレイスでSnap-inを検索・インストールできる
- Keyring Connectionsの設定方法を理解している
- AirSyncの概念と用途を説明できる

## 試してみる

マーケットプレイスでSlackまたはGitHub Snap-inをインストールしてみよう。

```
【Snap-inインストール演習】

□ 1. マーケットプレイスを開いた
□ 2. 利用したいSnap-inを検索した
□ 3. Snap-inの詳細ページで必要な権限を確認した
□ 4. Snap-inをインストールした
□ 5. Keyring Connectionを設定した
□ 6. テスト操作を行い、連携が動作することを確認した

インストールしたSnap-in: ______________
設定したConnection: ______________
確認した動作: ______________
```
