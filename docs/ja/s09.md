# 分析ダッシュボードを活用する

> "サポート・開発・運営の全チームのKPIを、1つのダッシュボードで可視化する。"

## 問題

KPIがバラバラのツールに散らばっていると、組織全体の状況を把握するために複数の画面を行き来する必要があります。

サポートの解決時間はZendeskで、開発の進捗はJiraで、顧客行動はHotjarで確認します。

これらを統合した週次レポートを作成するのに毎回数時間かかります。

## 解決策

DevRevのReports機能では、サポート・開発・運営の全データを1つのダッシュボードで可視化できます。

データが同一プラットフォーム上にあるため、クロスチームの分析が自然にできます。

## 仕組み

**ビルトインレポート**

DevRevにはあらかじめ用意されたレポートがあります。

| レポート種類 | 内容 |
|------------|------|
| Ticket Volume | 期間別のチケット数推移 |
| Resolution Time | チケット解決までの平均時間 |
| SLA Compliance | SLA達成率 |
| Issue Burndown | スプリント内のIssue消化状況 |
| Part Distribution | Part別のチケット・Issue分布 |

**カスタムダッシュボード**

ビルトインレポートでは不足する場合、カスタムダッシュボードを作成できます。

カスタムダッシュボードでは以下の要素を組み合わせます。

- **メトリクス**: 測定する対象（チケット数、解決時間、SLA達成率など）
- **フィルタ**: 条件（期間、Part、Priority、Assigneeなど）
- **グルーピング**: 集計軸（日別、週別、Part別、チーム別など）
- **表示形式**: グラフの種類（棒グラフ、折れ線、円グラフ、テーブルなど）

**Notebook（データ探索ツール）**

カスタムダッシュボードを作成する前段階として、DevRev には **Notebook** という SQL クエリインターフェースが用意されています。DuckDB 構文でデータを直接探索でき、クエリの検証後に Widget Builder でダッシュボードに変換する流れになります。

**クロスチーム分析の例**

DevRevならではの分析として、サポートデータと開発データを横断した分析ができます。

- 「チケットが最も多いPartと、そのPartの未解決Issue数」を1画面で確認
- 「SLA違反が多い時間帯と、その時間帯のサポート担当者の負荷」を相関分析
- 「リリース後にチケット数が減ったか」をリリースタイムラインとチケット数推移で確認

## 試してみる

チケット解決時間のレポートを1つ作成してみましょう。

```
【ダッシュボード作成演習】

□ 1. ビルトインレポートを3つ確認した
□ 2. カスタムダッシュボードを新規作成した
□ 3. チケット解決時間のメトリクスを追加した
□ 4. 期間フィルタを設定した（過去7日）
□ 5. Part別にグルーピングして表示した
□ 6. 表示形式を棒グラフに変更した

【作成したダッシュボードの設定メモ】
メトリクス: ______________
フィルタ:   ______________
グルーピング: ______________
表示形式:  ______________
```

次のセッションでは、ワークフローと自動化を設計します。プラットフォーム活用のレイヤーを終え、開発者・拡張のレイヤーに入ります。

---

## まとめ

このセッションを終えると、次のことができるようになっているはずです。

- ビルトインレポートの種類と用途を説明できる
- カスタムダッシュボードを作成し、フィルタとグルーピングを設定できる
- Notebookで DuckDB クエリを使ってデータを直接探索できる
- クロスチーム分析（サポート×開発データの横断分析）の観点を3つ以上挙げられる

---

## 用語

| 用語 | 説明 |
|------|------|
| **Reports** | DevRevのレポート機能。ビルトインレポートとカスタムダッシュボードを提供する |
| **カスタムダッシュボード** | メトリクス・フィルタ・グルーピング・表示形式を組み合わせて作成する独自のダッシュボード |
| **Notebook** | DuckDB構文でデータを直接探索できるSQLクエリインターフェース。ダッシュボード作成前の検証に使う |
| **メトリクス** | 測定する対象（チケット数、解決時間、SLA達成率など） |
| **Widget Builder** | Notebookで検証したクエリをダッシュボードのウィジェットに変換するツール |
| **クロスチーム分析** | サポートデータと開発データを横断して行う分析。同一プラットフォームだからこそ可能 |
