# はじめてのDevRev：セットアップと基本操作

> "最初の30分で全体像をつかめば、その後の学習スピードが変わる。"

## 問題

DevRevに初めてログインすると、左サイドバーには**見覚えのない名前のメニュー**が並んでいる。

- 「Tickets」を探すと → **Product Assist Tracker**
- 「Home」を探すと → **Updates**
- 「Backlog」を探すと → **Triage**

JiraやZendeskに慣れていると、この命名の違いだけで混乱する。

さらに、よく使うメニューの多くはデフォルトの左サイドバーに表示されていない。**Exploreから自分でピン留めして追加する**必要があるが、その方法はUI上で説明されない。

## 解決策

最初にやるべきことは4ステップに絞られる。

1. **ワークスペースの作成**: DevRevでは「ワークスペース」が組織の基本単位。まずこれを作成する
2. **プロフィール設定**: 自分のアカウント情報を設定し、チームメンバーとして認識される状態にする
3. **左サイドバーのカスタマイズ**: Exploreからよく使うメニューをピン留めして、ナビゲーションを自分仕様にする
4. **基本ナビゲーションの把握**: 左サイドバーの構造を理解し、主要な画面に迷わずアクセスできるようにする

この4つを最初の30分で済ませることが、学習加速の鍵になる。

## 仕組み

### 左サイドバーのデフォルト構成

DevRevにログインすると、左サイドバーには最初から多くの項目が表示されている。

**固定メニュー（セクションなし）**

| メニュー | 対応するイメージ | 実際の役割 |
|---------|----------------|-----------|
| **Updates** | ≒ Home / Notification | 自分宛ての更新通知・アクティビティフィード |
| **My Tasks** | ≒ 自分のタスク一覧 | 自分に割り当てられた作業アイテム |
| **Explore** | ナビゲーション管理 | ここからメニューをピン留め・カスタマイズする |

**Workセクション（開発寄り）**

| メニュー | 対応するJira概念 |
|---------|----------------|
| Issues | バックログのIssue |
| Triage | 未分類・新着のIssue振り分け |
| Now, Next, Later | 優先度別のロードマップビュー |
| Sprint Boards | スプリントボード |
| Dev360 | 開発メトリクスダッシュボード |

**Productセクション**

| メニュー | 役割 |
|---------|------|
| Trails | Parts階層（製品/機能/コンポーネント）の関係性をグラフで可視化するビュー |
| Parts | プロダクト構造の定義（製品/機能/コンポーネントのライフサイクル管理） |
| Roadmap | ロードマップビュー |
| Release Tracker | リリース管理 |

**Customerセクション（サポート寄り）**

| メニュー | 役割 |
|---------|------|
| Product Assist Tracker | 顧客フィードバックをもとに機能強化（エンハンスメント）を追跡・優先順位付けするビュー |
| Accepted Ticket Tracker | 将来的なロードマップへの組み込みが確定した機能リクエストを管理するビュー |

**Dataセクション**

| メニュー | 役割 |
|---------|------|
| Notebook | SQLクエリをテスト・検証するデータ探索ツール（ダッシュボード作成前の確認に使う） |

> **ポイント**：「Tickets」「Home」「Backlog」といった馴染みの名前はデフォルトのサイドバーには表示されていない。DevRev固有の命名体系に慣れることが最初のハードルになる。

### 左サイドバーのカスタマイズ方法（Exploreからピン留め）

左サイドバーへのメニュー追加は「Explore」から行う。

1. 左サイドバーの下部にある **「Explore」** をクリック
2. 追加したいビューにマウスをホバーする
3. 表示される **ピンアイコン（📌）** をクリック
4. 左サイドバーにそのビューが追加される

ピンを外すには、同じビューにホバーして **アンピンアイコン** をクリックする。

#### Exploreの構成

| カテゴリ | 内容 |
|---------|------|
| **Stock Views** | DevRevが提供するデフォルトビュー（My Issues、My Meetings など） |
| **My Views** | 自分が作成したカスタムビュー |
| **Shared** | 組織やチームと共有されたビュー |

> **Recentsセクション**：ピン留めしていないビューのうち、最近開いた5件が自動的に左サイドバーに表示される。Exploreから開いたり、共有リンクでアクセスしたビューが対象。

### このコースで使うメニューをExploreから追加する

デフォルトの左サイドバーには含まれていないが、このコース（s01〜s14）を通じてよく使うメニューを先にまとめて追加しておく。Exploreの検索バーにメニュー名を入力し、Stock Viewsに表示されたビューをホバーして **ピンアイコン** をクリックすれば追加できる。

| メニュー名 | 検索ワード | 使うセッション | 用途 |
|-----------|----------|--------------|------|
| **Tickets** | ticket | s04〜 | サポートチケットの管理 |
| **Conversations** | conv | s06〜 | PLuGのチャット対応 |
| **Inbox** | inbox | s06〜 | チャット・メールなど顧客メッセージを一箇所で確認 |
| **Accounts** | account | s03〜 | 顧客企業の管理（CRM） |
| **Contacts** | contact | s03〜 | 顧客の担当者・個人の管理（CRM） |

#### Ticketsは自分のチケットだけ表示するフィルタを保存する

追加したTicketsビューは全チケットが表示される。自分に割り当てられたチケットだけ見たい場合は、フィルタを保存する。

1. Ticketsビューを開く
2. フィルタバーから **「Assignee」→ 自分のアカウント** を選択
3. 画面右上の **「Save」** をクリック

### その他の基本機能

DevRevには検索機能とキーボードショートカットが用意されている。画面上部の検索バーからチケット、Issue、Partを横断検索できる。

ワークスペース作成時には以下の情報を設定する。

| 設定項目 | 説明 |
|---------|------|
| ワークスペース名 | 組織名またはプロジェクト名 |
| タイムゾーン | SLAポリシーの基準時刻に影響する |
| デフォルト言語 | UIとテンプレートの言語設定 |

## このセッションで学ぶこと

**始める前の状態：**
- DevRevにアカウントを作ったが、ログイン後に何をすればいいかわからない
- 左サイドバーにメニューはあるが、見知らぬ名前ばかりで何がどこにあるかわからない
- Exploreからピン留めしてナビゲーションをカスタマイズする方法を知らない
- 検索やショートカットの存在を知らない

**このセッション後の状態：**
- ワークスペースを作成し、プロフィール設定が完了している
- Exploreからよく使うビューをピン留めし、自分仕様の左サイドバーを構成できる
- 左サイドバーの4セクション（Work / Product / Customer / Data）の役割を説明できる
- 検索バーを使って目的のオブジェクトにすばやくアクセスできる

## 試してみる

DevRevのワークスペースを実際に作成し、ナビゲーションをカスタマイズしてみよう。

```
【セットアップ・チェックリスト】

□ 1. DevRevアカウントを作成した
□ 2. ワークスペースを作成した（名前: __________）
□ 3. タイムゾーンを設定した
□ 4. プロフィール（名前・アバター）を設定した
□ 5. 左サイドバーの「Explore」を開いた
□ 6. ExploreでTicketsを検索し、Stock Viewsからピン留めした
□ 7. TicketsビューでAssigneeフィルタを設定し、Saveした
□ 8. ExploreでConversationsを検索し、Stock Viewsからピン留めした
□ 9. ExploreでInboxを検索し、Stock Viewsからピン留めした
□ 10. ExploreでAccountsを検索し、Stock Viewsからピン留めした
□ 11. ExploreでContactsを検索し、Stock Viewsからピン留めした
□ 12. チームメンバーを1人招待した
□ 13. 検索バーで何か検索してみた
```
