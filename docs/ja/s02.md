# はじめてのDevRev：セットアップと基本操作

> "最初の30分で全体像をつかめば、その後の学習スピードが変わる。"

## 問題

DevRevに初めてログインすると、左サイドバーに多数のメニュー項目が並んでおり、どこから手をつければいいかわからない。ドキュメントは英語中心で、設定項目も多い。結果として、最初の体験で挫折してしまうケースが少なくない。

特にJiraやZendeskに慣れた人は、DevRevのナビゲーション構造が異なるため混乱しやすい。

## 解決策

最初にやるべきことは3ステップに絞られる。

1. **ワークスペースの作成**: DevRevでは「ワークスペース」が組織の基本単位。まずこれを作成する
2. **プロフィール設定**: 自分のアカウント情報を設定し、チームメンバーとして認識される状態にする
3. **基本ナビゲーションの把握**: 左サイドバーの構造を理解し、主要な画面に迷わずアクセスできるようにする

この3つを最初の30分で済ませることが、学習加速の鍵になる。

## 仕組み

DevRevのインターフェースは、左サイドバーを中心に構成されている。

主要なナビゲーション要素は以下の通り。

- **Home**: ダッシュボードとアクティビティフィード
- **Inbox**: 自分に割り当てられたチケットやIssueの一覧
- **Tickets**: サポートチケットの管理画面（Support App）
- **Issues**: 開発課題の管理画面（Build App）
- **Parts**: プロダクト構造の定義と管理
- **Settings**: ワークスペース全体の設定

DevRevには検索機能とキーボードショートカットが用意されている。画面上部の検索バーからチケット、Issue、Partを横断検索できる。

ワークスペース作成時には以下の情報を設定する。

| 設定項目 | 説明 |
|---------|------|
| ワークスペース名 | 組織名またはプロジェクト名 |
| タイムゾーン | SLAポリシーの基準時刻に影響する |
| デフォルト言語 | UIとテンプレートの言語設定 |

## このセッションで学ぶこと

**始める前の状態：**
- DevRevにアカウントを作ったが、ログイン後に何をすればいいかわからない
- 左サイドバーのメニュー構造が把握できていない
- 検索やショートカットの存在を知らない

**このセッション後の状態：**
- ワークスペースを作成し、プロフィール設定が完了している
- 左サイドバーの主要メニュー（Home / Inbox / Tickets / Issues / Parts）の役割を説明できる
- 検索バーを使って目的のオブジェクトにすばやくアクセスできる

## 試してみる

DevRevのワークスペースを実際に作成し、チームメンバーを1人招待してみよう。

```
【セットアップ・チェックリスト】

□ 1. DevRevアカウントを作成した
□ 2. ワークスペースを作成した（名前: __________）
□ 3. タイムゾーンを設定した
□ 4. プロフィール（名前・アバター）を設定した
□ 5. チームメンバーを1人招待した
□ 6. 左サイドバーの各メニューをクリックして確認した
□ 7. 検索バーで何か検索してみた
```
