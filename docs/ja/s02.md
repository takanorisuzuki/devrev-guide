# はじめてのDevRev：セットアップと基本操作

> "最初に「どこに何があるか」をつかむ。割り当てられたワークスペースにログインし、ナビゲーション設定を済ませて、以降のセッションを迷わず進める。"

## 問題

DevRevに初めてログインすると、左サイドバーには**見覚えのない名前のメニュー**が並んでいる。

たとえば、Jira/Zendeskの感覚で探すと、次のように**「やりたいこと」とメニュー名が一致しない**ことがある（組織の設定やピン留め状況によって表示は変わる）。

| 探しているもの（Jira/Zendeskでの呼び方の例） | DevRev上のメニュー名（例） | それは何か |
|---|---|---|
| 機能要望（顧客フィードバックの一覧） | **Product Assist Tracker** | 顧客フィードバックをもとに機能強化（エンハンスメント）を追跡・優先順位付けするビュー |
| Home（ホーム/通知） | **Updates** | 自分宛ての更新通知・アクティビティフィード |
| Backlog（未処理の山） | **Triage** | 新着・未分類の作業を振り分ける入口 |

JiraやZendeskに慣れていると、この命名の違いだけで混乱する。

さらに、よく使うメニューの多くはデフォルトの左サイドバーに表示されていない。**Exploreから自分でピン留めして追加する**必要があるが、その方法はUI上で説明されない。

## 解決策

最初にやるべきことは次の5つに絞られる。

1. **割り当てられたワークスペースにログイン**: 本ガイドでは、DevRev社員（または管理者）から割り当てられたワークスペースにログインして学習する前提とする（一般ユーザーが新規作成する想定ではない）
2. **プロフィール設定**: 自分のアカウント情報を設定し、チームメンバーとして認識される状態にする
3. **左サイドバーのカスタマイズ**: Exploreからよく使うメニューをピン留めして、ナビゲーションを自分仕様にする
4. **基本ナビゲーションの把握**: 左サイドバーの構造を理解し、主要な画面に迷わずアクセスできるようにする
5. **Settings（組織設定）の地図を頭に入れる**: 以降のセッションでは `Settings > …` のパスが頻出する。左メニューのカテゴリだけ先に押さえておくと迷子にならない

この5つを最初のひととおりで済ませることが、学習加速の鍵になる。

## 仕組み

### 左サイドバーのデフォルト構成

DevRevにログインすると、左サイドバーには最初から多くの項目が表示されている。

**固定メニュー（セクションなし）**

| メニュー | 対応するイメージ | 実際の役割 |
|---------|----------------|-----------|
| **Updates** | ≒ Home / Notification | 自分宛ての更新通知・アクティビティフィード |
| **My Tasks** | ≒ 自分のタスク一覧 | 自分に割り当てられた作業アイテム |
| **Explore** | ナビゲーション管理 | ここからメニューをピン留め・カスタマイズする |

**Workセクション（開発寄り）**

| メニュー | 対応するJira概念 |
|---------|----------------|
| Issues | バックログのIssue |
| Triage | 未分類・新着のIssue振り分け |
| Now, Next, Later | 優先度別のロードマップビュー |
| Sprint Boards | スプリントボード |
| Dev360 | 開発メトリクスダッシュボード |

**Productセクション**

| メニュー | 役割 |
|---------|------|
| Trails | Parts階層（製品/機能/コンポーネント）の関係性をグラフで可視化するビュー |
| Parts | プロダクト構造の定義（製品/機能/コンポーネントのライフサイクル管理） |
| Roadmap | ロードマップビュー |
| Release Tracker | リリース管理 |

**Customerセクション（サポート寄り）**

| メニュー | 役割 |
|---------|------|
| Product Assist Tracker | 顧客フィードバックをもとに機能強化（エンハンスメント）を追跡・優先順位付けするビュー |
| Accepted Ticket Tracker | 将来的なロードマップへの組み込みが確定した機能リクエストを管理するビュー |

**Dataセクション**

| メニュー | 役割 |
|---------|------|
| Notebook | SQLクエリをテスト・検証するデータ探索ツール（ダッシュボード作成前の確認に使う） |

> **ポイント**：「Tickets」「Home」「Backlog」といった馴染みの名前はデフォルトのサイドバーには表示されていない。DevRev固有の命名体系に慣れることが最初のハードルになる。

### 左サイドバーのカスタマイズ方法（Exploreからピン留め）

左サイドバーへのメニュー追加は「Explore」から行う。

1. 左サイドバーの下部にある **「Explore」** をクリック
2. 追加したいビューにマウスをホバーする
3. 表示される **ピンアイコン（📌）** をクリック
4. 左サイドバーにそのビューが追加される

ピンを外すには、同じビューにホバーして **アンピンアイコン** をクリックする。

#### Exploreの構成

| カテゴリ | 内容 |
|---------|------|
| **Stock Views** | DevRevが提供するデフォルトビュー（My Issues、My Meetings など） |
| **My Views** | 自分が作成したカスタムビュー |
| **Shared** | 組織やチームと共有されたビュー |

> **Recentsセクション**：ピン留めしていないビューのうち、最近開いた5件が自動的に左サイドバーに表示される。Exploreから開いたり、共有リンクでアクセスしたビューが対象。

### このコースで使うメニューをExploreから追加する

デフォルトの左サイドバーには含まれていないが、このコース（s01〜s14）を通じてよく使うメニューを先にまとめて追加しておく。Exploreの検索バーにメニュー名を入力し、Stock Viewsに表示されたビューをホバーして **ピンアイコン** をクリックすれば追加できる。

| メニュー名 | 検索ワード | 使うセッション | 用途 |
|-----------|----------|--------------|------|
| **Tickets** | ticket | s04〜 | サポートチケットの管理 |
| **Conversations** | conv | s06〜 | PLuGのチャット対応 |
| **Inbox** | inbox | s06〜 | チャット・メールなど顧客メッセージを一箇所で確認 |
| **Accounts** | account | s03〜 | 顧客企業の管理（CRM） |
| **Contacts** | contact | s03〜 | 顧客の担当者・個人の管理（CRM） |

#### Ticketsは自分のチケットだけ表示するフィルタを保存する

追加したTicketsビューは全チケットが表示される。自分に割り当てられたチケットだけ見たい場合は、フィルタを保存する。

1. Ticketsビューを開く
2. フィルタバーから **「Assignee」→ 自分のアカウント** を選択
3. 画面右上の **「Save」** をクリック

### その他の基本機能

DevRevには検索機能とキーボードショートカットが用意されている。画面上部の検索バーからチケット、Issue、Partを横断検索できる。

### Settings（組織設定）の全体像

メインの左サイドバーとは別に、**組織単位の設定は Settings に集約**されている。画面右上のプロフィールメニューなどから **Settings** を開くと、URL は `https://app.devrev.ai/<組織スラッグ>/settings/...` の形式になる。

ここでは各項目の設定手順までは扱わない（個別のセッションで触れる）。**左メニューにどんなカテゴリがあるか**だけ先に把握しておくと、s04 以降の「Settings > Support > …」などの指示にすぐ辿り着ける。

| カテゴリ（左メニュー） | ざっくり何があるか | 主にこのコースのどこで触れるか |
|----------------------|-------------------|-------------------------------|
| **General** | 組織名、データリージョン、会計年度・タイムゾーン、組織削除など | 初期セットアップ時（Owner/Admin）と、[s08](/ja/s08) 前後 |
| **Account** | アカウントまわりの組織設定 | 必要に応じて |
| **Language and Region** | 言語・地域 | [s05](/ja/s05)（言語・ポータル）と関連 |
| **Security** | セキュリティポリシー | 運用ポリシーに応じて |
| **Notifications** | 通知 | 運用に応じて |
| **Support** | Org Scheduler、SLA / OLA、チケット設定、ナレッジベース、Q&A など | [s04](/ja/s04)、[s05](/ja/s05) |
| **PLuG & Portal** | PLuG（チャット・検索・トークンなど）、ポータル、セッションリプレイ | [s06](/ja/s06)、[s05](/ja/s05) |
| **User Management** | グループ、ロール、ユーザー、招待 | [**s08**](/ja/s08)（ロールと権限） |
| **Customer Management** | 顧客向けグループ・ロール | 必要に応じて |
| **Integrations** | Snap-ins、AirSync | [s05](/ja/s05)、[s11](/ja/s11)、[s12](/ja/s12) など |
| **Customization** | 下位に Commands、Object Customization、Tags、Templates など | Object Customization は [architecture](/ja/architecture)、API・オブジェクトは [s12](/ja/s12)。テンプレートやタグは運用設計で |
| **Billing** | Invoices and Payments（請求・支払い） | Owner / Admin 向け |
| **Workflows** | ワークフロー | [s10](/ja/s10) |
| **Agents** | エージェント設計の入口（Agent Studio） | [s14](/ja/s14)（Agent Studio） |
| **Computer** | プラットフォームの Computer に関する設定 | [s01](/ja/s01)、[s14](/ja/s14) の文脈 |
| **Jobs** | ジョブ実行の確認・管理 | 運用で触れることがある |

> **ポイント**: 製品アップデートでラベルや並びは変わることがある。迷ったら Settings の検索バー（ある場合）や公式ドキュメントと併用する。社内向けのメニュー（組織によっては表示されるもの）は学習ガイドでは扱わない。

初期セットアップ（通常は Owner/Admin、または DevRev 側）では、たとえば以下の情報が設定される。

| 設定項目 | 説明 |
|---------|------|
| ワークスペース名 | 組織名またはプロジェクト名 |
| タイムゾーン | SLAポリシーの基準時刻に影響する |
| デフォルト言語 | UIとテンプレートの言語設定 |

## このセッションで学ぶこと

**始める前の状態：**
- DevRevにアカウントを作ったが、ログイン後に何をすればいいかわからない
- 左サイドバーにメニューはあるが、見知らぬ名前ばかりで何がどこにあるかわからない
- Exploreからピン留めしてナビゲーションをカスタマイズする方法を知らない
- 検索やショートカットの存在を知らない

**このセッション後の状態：**
- 割り当てられたワークスペースにログインし、プロフィール設定が完了している
- Exploreからよく使うビューをピン留めし、自分仕様の左サイドバーを構成できる
- 左サイドバーの4セクション（Work / Product / Customer / Data）の役割を説明できる
- Settings を開き、左メニューのカテゴリ（General / Support / PLuG & Portal / User Management / Integrations など）がどこに相当するか説明できる
- 検索バーを使って目的のオブジェクトにすばやくアクセスできる

## 試してみる

割り当てられたワークスペースにログインし、ナビゲーションをカスタマイズしてみよう。

```
【セットアップ・チェックリスト】

□ 1. DevRevアカウントを作成した
□ 2. 割り当てられたワークスペースにログインした（URL: __________）
□ 3. タイムゾーン設定を確認した（必要なら Owner/Admin に依頼する）
□ 4. プロフィール（名前・アバター）を設定した
□ 5. 左サイドバーの「Explore」を開いた
□ 6. ExploreでTicketsを検索し、Stock Viewsからピン留めした
□ 7. TicketsビューでAssigneeフィルタを設定し、Saveした
□ 8. ExploreでConversationsを検索し、Stock Viewsからピン留めした
□ 9. ExploreでInboxを検索し、Stock Viewsからピン留めした
□ 10. ExploreでAccountsを検索し、Stock Viewsからピン留めした
□ 11. ExploreでContactsを検索し、Stock Viewsからピン留めした
□ 12. チームメンバーを1人招待した
□ 13. Settings を開き、左メニューのカテゴリ（General / Support / PLuG & Portal / User Management / Integrations など）を一通り目を通した
□ 14. 検索バーで何か検索してみた
```
